    <action>
      Implements D-09 (server-ci.yml is a NEW separate workflow file — NOT added to rust-core-ci.yml).

      Create `.github/workflows/server-ci.yml` with the following content:

      ```yaml
      name: Server CI

      # Builds and tests the Goose server ingest service. The conftest.py fixture
      # manages a throwaway TimescaleDB container via docker run — no services: block
      # needed because Docker daemon is available on ubuntu-latest runners.

      on:
        push:
          branches: ["main"]
          paths:
            - "server/**"
            - ".github/workflows/server-ci.yml"
        pull_request:
          paths:
            - "server/**"
            - ".github/workflows/server-ci.yml"
        workflow_dispatch:

      defaults:
        run:
          working-directory: server/ingest

      jobs:
        pytest:
          name: Server pytest suite
          runs-on: ubuntu-latest
          steps:
            - name: Check out repository
              uses: actions/checkout@v4

            - name: Set up Python 3.12
              uses: actions/setup-python@v5
              with:
                python-version: "3.12"

            - name: Cache pip
              uses: actions/cache@v4
              with:
                path: ~/.cache/pip
                key: ${{ runner.os }}-pip-${{ hashFiles('server/ingest/requirements-dev.txt') }}
                restore-keys: |
                  ${{ runner.os }}-pip-

            - name: Install dependencies
              run: pip install -r requirements-dev.txt

            - name: Verify Docker is available
              run: docker info

            - name: Run pytest
              run: pytest tests/ -v --tb=short
      ```

      Key decisions:
      - `defaults: run: working-directory: server/ingest` avoids repeating `cd server/ingest` in every step
      - `python-version: "3.12"` matches ubuntu-latest pre-installed version for consistency
      - No `services:` block — `conftest.py` manages the TimescaleDB container via `docker run` with a random ephemeral port (`-P` flag) and 60-second readiness polling
      - `pytest tests/ -v --tb=short` — `tests/` relative to `server/ingest/` working directory
      - `workflow_dispatch:` allows manual trigger for debugging
      - Trigger path `server/**` catches any server file change; `server-ci.yml` change itself re-triggers
    </action>
    <acceptance_criteria>
      - `.github/workflows/server-ci.yml` exists
      - File contains `on: push: paths: - "server/**"` and `on: pull_request: paths: - "server/**"`
      - File contains `uses: actions/setup-python@v5` with `python-version: "3.12"`
