## Summary

Closed gap WEAR-01/WEAR-03 and fixed CR-02. Three commits:

1. **Root cause fix (capture_import.rs):** HR monitor 0x2A37 GATT frames were rejected by `parse_frame` (requires 0xAA start byte) and never reached `decoded_frames`. Added `DeviceType::HrMonitor` branch in `import_captured_frame_timed` that bypasses `parse_frame`, constructs a pseudo-`ParsedFrame` with `payload_hex = hex::encode(raw_bytes)` and both CRC flags = true, then inserts via `store.insert_decoded_frame`.

2. **Upload bridge integration (bridge.rs):** Added `if frame.device_type == "HR_MONITOR"` branch in `upload_get_recent_decoded_streams_bridge` after the WHOOP match block. Decodes `frame.payload_hex` via `hex::decode`, calls `parse_hr_measurement`, formats result as `{"ts": f64, "bpm": u16, "rr_intervals": [f64]}` pushed to `hr[]` (D-01/D-02). Added `unix_from_iso8601` helper (no chrono dependency, D-03 intent). Fixed CR-02 device_id filter via `store.raw_evidence(evidence_id).device_model` comparison.

3. **Test timestamp fix (bridge_tests.rs):** HR monitor integration tests used future `captured_at` timestamps (10am UTC, but tests ran at midnight). Fixed to use 2020 past dates so `decoded_frames_between` query (`captured_at < chrono_now()`) includes the test frames.
