Add a new `android-build` job to the existing `.github/workflows/rust-core-ci.yml` workflow. The job installs cargo-ndk 4.1.2 and Android NDK r29 (via `nttld/setup-ndk@v1`), then runs `cargo ndk -t arm64-v8a build --release --lib`.     <action>
      Implements D-07 (cargo-ndk 4.1.2, NDK r29) and D-08 (android job added to existing rust-core-ci.yml — NOT a new file). Build target is aarch64-linux-android only (D-04).

      In `.github/workflows/rust-core-ci.yml`:

      After the existing `build-test:` job definition (after its last step), add a new top-level job `android-build:` at the same indentation level under `jobs:`:

      ```yaml
        android-build:
          name: Android cross-compile (aarch64)
          runs-on: ubuntu-latest
          defaults:
            run:
              working-directory: Rust/core
          steps:
            - name: Check out repository
              uses: actions/checkout@v4

            - name: Set up Rust toolchain
              run: |
                rustup toolchain install stable --profile minimal
                rustup default stable
                rustup target add aarch64-linux-android
                rustc --version
                cargo --version

            - name: Set up Android NDK r29
              uses: nttld/setup-ndk@v1
              with:
                ndk-version: r29
                add-to-path: true

            - name: Cache cargo registry and build artifacts
              uses: actions/cache@v4
              with:
                path: |
                  ~/.cargo/registry
                  ~/.cargo/git
                key: ${{ runner.os }}-cargo-android-${{ hashFiles('Rust/core/Cargo.lock') }}
                restore-keys: |
                  ${{ runner.os }}-cargo-android-

            - name: Install cargo-ndk
              run: cargo install cargo-ndk --version 4.1.2 --locked

            - name: Build for aarch64-linux-android
              run: cargo ndk -t arm64-v8a build --release --lib
      ```

      Key decisions:
      - `--lib` builds only the library target — skips the ~20 binary targets which may not cross-compile
      - `--release` uses the release profile (which already has panic = "abort")
      - Cache key `cargo-android-` is distinct from the existing `cargo-` prefix to avoid cache poisoning between host and Android target artifact caches
      - The job does NOT include `cargo test` — no Android runtime is available to run tests
      - The job does NOT depend on `build-test` — both jobs run in parallel (saves CI time)
      - `nttld/setup-ndk@v1` sets ANDROID_NDK_HOME automatically, which cargo-ndk reads
    </action>
