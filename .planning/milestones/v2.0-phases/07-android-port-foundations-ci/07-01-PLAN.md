<objective>
Prepare the Rust crate for Android cross-compilation by (1) moving `tungstenite` from `[dependencies]` to `[target.'cfg(not(target_os = "android"))'.dependencies]`, (2) adding the `jni` crate under `[target.'cfg(target_os = "android")'.dependencies]`, and (3) cfg-gating the `debug_ws_server` module declaration in `lib.rs` so it is excluded on Android (it directly imports `tungstenite`).

After this plan, `cargo check --target aarch64-linux-android` must succeed for the library (ignoring binary targets).
</objective>

<threat_model>
No user-facing attack surface in this plan. The changes are build-configuration only.

- T-01: Incorrect cfg expression could include tungstenite on Android → would fail `cargo build` on Android target; caught immediately by the CI job in Plan 07-03.
- T-02: Removing tungstenite from `[dependencies]` on non-Android builds → compile error on host; checked by `cargo build --locked` step in acceptance criteria.
Severity: LOW (compile-time, not runtime).
Block threshold: HIGH. No runtime security threats in this plan.
</threat_model>
