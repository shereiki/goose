      .padding(.top, 36)
      .padding(.bottom, 28)

  .navigationBarTitleDisplayMode(.inline)
  .toolbarBackground(.hidden, for: .navigationBar)
```

### Scanning State Sub-Hierarchy

```
HRMonitorScanList
  VStack(alignment: .leading, spacing: 12)
    Text("DISCOVERED")                 ← section label, deviceLabelFont, secondaryText
    [if discoveredHRDevices.isEmpty]
      Text("No devices found")         ← deviceBodyFont, mutedText
    [else]
      VStack(spacing: 0)
        ForEach(discoveredHRDevices)
          HRMonitorDeviceRow           ← tap → sheet
            HStack(spacing: 12)
              VStack(alignment: .leading, spacing: 4)
                Text(device.name)      ← deviceBodyFont.weight(.black), devicePrimaryText
                Text("... · ... dBm")  ← 12pt semibold, mutedText
              Spacer()
              [if connecting to this device]
                ProgressView()         ← .scaleEffect(0.8), tinted secondaryText
            .padding(.vertical, 13)
            .contentShape(Rectangle())
          .overlay(alignment: .bottom) Rectangle dividerColor height 1
```

```
HRMonitorConnectedPanel
  VStack(alignment: .leading, spacing: 22)
    HRLiveBPMDisplay
      VStack(alignment: .leading, spacing: 7)
        Text("HEART RATE")             ← deviceLabelFont, secondaryText

Exceptions: ProgressView spinner inline on row — no padding override, inherits row padding.

## Header Design

`HRMonitorHeader` mirrors `DeviceConnectionHeader` from DeviceView.swift exactly:

```
HStack(alignment: .bottom, spacing: 16)
  VStack(alignment: .leading, spacing: 7)
    Text(statusText)                   ← deviceLabelFont; connectedGreen if connected, secondaryText otherwise
    Text(deviceDisplayName.uppercased()) ← .system(size: 26, weight: .black); devicePrimaryText
  .frame(maxWidth: .infinity, alignment: .leading)
```

Status text values:
- Scanning state: `"SCANNING"` — `secondaryText`
- Connecting state: `"CONNECTING"` — `secondaryText`
- Connected state: `"CONNECTED"` — `connectedGreen`
- Bluetooth off/unauthorized: `"BLUETOOTH OFF"` or `"NOT AUTHORISED"` — `disconnectedRed`

Device display name:
- Scanning state: `"HR Monitor"` (placeholder)
- Connected state: `hrMonitorManager.connectedDeviceName ?? "HR Monitor"`

No "LAST SYNC" column in the right side (no sync concept for HR monitors). The HStack has only the left VStack, with `maxWidth: .infinity`.

## State Machine

Four mutually exclusive UI states, determined by `ble.hrConnectionState` and `ble.bluetoothState`:

```
State 1 — BLUETOOTH_UNAVAILABLE
  Condition: bluetoothState != "poweredOn" (e.g. "unauthorized", "poweredOff")
  Header: status = "BLUETOOTH OFF" (disconnectedRed) or "NOT AUTHORISED" (disconnectedRed)
  Body: single centered message, no scan list
  Copy: see Copywriting section

State 2 — SCANNING
  Condition: hrConnectionState == "disconnected" (and Bluetooth on)
  Header: status = "SCANNING" (secondaryText), name = "HR Monitor"
  Body: HRMonitorScanList with live discoveredHRDevices
  Lifecycle: startHRMonitorScan() on onAppear; stopHRMonitorScan() on onDisappear

State 3 — CONNECTING
  Condition: hrConnectionState == "connecting" (during connect attempt)
  Header: status = "CONNECTING" (secondaryText)
  Body: scan list still visible; target device row shows inline ProgressView
  Sheet: dismissed immediately after user taps "Connect"

State 4 — CONNECTED
  Condition: hrConnectionState == "connected"
  Header: status = "CONNECTED" (connectedGreen), name = connectedDeviceName
  Body: HRMonitorConnectedPanel (BPM display, reconnect state, Disconnect button)
  Lifecycle: skip scan on onAppear; after disconnect → return to State 2
```

State variable to add to `GooseBLEClient`: `@Published var discoveredHRDevices: [GooseDiscoveredDevice]` mirroring `hrMonitorManager.discoveredHRDevices` (promoted state pattern — CONTEXT.md Claude's Discretion).

Connecting device tracking: `@State private var connectingDeviceID: UUID?` in `HRMonitorView` — set when user taps "Connect" in sheet, cleared when `hrConnectionState` becomes "connected" or on error.

## Interaction Flows

### Flow 1: Scan and Connect

1. User navigates More > HR Monitor
2. `onAppear`: if `hrConnectionState != "connected"`, call `ble.startHRMonitorScan()`
3. Scan list populates as `discoveredHRDevices` updates (live, sorted by RSSI descending)
4. User taps a device row → sheet appears (`.sheet(isPresented:)`)
5. Sheet shows device name (uppercased), RSSI, and "Connect" button
6. User taps "Connect" → `ble.connectHRMonitor(device)` called; `connectingDeviceID = device.id`; sheet dismissed
7. Target row shows `ProgressView()` inline (trailing position in HStack)
8. When `hrConnectionState == "connected"`: `connectingDeviceID = nil`; view switches to connected state

### Flow 2: Disconnect

1. User taps "Disconnect" in connected state
2. No confirmation sheet — direct action (disconnect is recoverable; device can be reconnected by rescanning)
3. Call sequence: `ble.stopHRMonitorScan()` + `hrMonitorManager.hrStopReconnect()` + cancel peripheral connection
4. `hrConnectionState` becomes "disconnected" → view switches back to scanning state
5. `ble.startHRMonitorScan()` auto-restarts scan

### Flow 3: Reconnect State Display

1. After unexpected disconnect, `hrReconnectState` becomes non-idle (e.g. "reconnecting (attempt 3/10)")
2. Connected panel shows `hrReconnectState` text in `disconnectedRed` below BPM
3. If reconnect succeeds → `hrReconnectState` = "idle" → text disappears
4. If reconnect fails ("failed after 10 attempts") → text shown permanently until manual disconnect/reconnect

### Flow 4: Sheet Dismissal Without Connecting

1. User taps device → sheet appears
2. User swipes sheet down (drag indicator visible) — no action taken
3. Sheet dismissed, scan list resumes

## State Observation Contract

`HRMonitorView` observes `model.ble` via `@ObservedObject`:

| Property | Used For |
|----------|---------|
| `ble.discoveredHRDevices` | Scan list population (requires `@Published` promotion — Claude's Discretion) |
| `ble.hrConnectionState` | State machine: "disconnected" / "connecting" / "connected" |
| `ble.liveHeartRateBPM` | BPM display in connected state |
| `ble.hrReconnectState` | Reconnect status text in connected state |
| `ble.bluetoothState` | Bluetooth availability gate (State 1) |

View-local state:
- `@State private var connectingDeviceID: UUID?` — tracks which device is mid-connection
- `@State private var selectedDevice: GooseDiscoveredDevice?` — drives sheet presentation
