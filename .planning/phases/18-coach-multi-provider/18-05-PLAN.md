<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → Keychain/UserDefaults | API keys (SecureField) and URLs entered in the settings sheet |
| settings sheet → provider auth | sign-in/sign-out actions mutate stored credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-14 | Information Disclosure | API key entry fields | mitigate | API keys use `SecureField` (masked); on save the field clears and the key is written to Keychain only, never displayed again (UI-SPEC Interaction Contracts) |
| T-18-15 | Tampering | Custom URL input | mitigate | Inline validation requires https:// for non-local before save; reuses RemoteServerURLValidator (V5) |
| T-18-16 | Elevation/Destruction | Sign-out / remove-key actions | mitigate | Both gated behind `.confirmationDialog` with explicit destructive confirmation before deleting credentials |
| T-18-SC | Tampering | npm/pip/cargo installs | mitigate | Zero external packages (D-08); native SwiftUI only |
</threat_model>
