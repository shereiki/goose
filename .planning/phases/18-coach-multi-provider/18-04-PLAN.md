<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → accounts.google.com (WKWebView) | User credentials entered on Google's page; redirect carries auth code |
| app → oauth2.googleapis.com | PKCE code + refresh token exchanged for access token |
| app → generativelanguage.googleapis.com | Bearer token + chat content cross to Google over TLS |
| app → Keychain | OAuth token struct stored and read |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-10 | Spoofing | OAuth redirect interception | mitigate | PKCE (S256 code_challenge via CryptoKit) + `state` parameter checked; WKNavigationDelegate verifies `url.scheme == "gooseswift"` |
| T-18-11 | Information Disclosure | OAuth tokens | mitigate | Stored only in Keychain (com.goose.swift.gemini) with `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly`; client_id (non-secret) in UserDefaults |
| T-18-12 | Information Disclosure | Token in memory after signOut | mitigate | signOut() deletes Keychain entry AND clears in-memory token reference |
| T-18-13 | Tampering | TLS to Google endpoints | accept | URLSession ATS enforces https; ephemeral session for token exchange disables cookies |
| T-18-SC | Tampering | npm/pip/cargo installs | mitigate | Zero external packages (D-08); no Google Sign-In SDK |
</threat_model>
